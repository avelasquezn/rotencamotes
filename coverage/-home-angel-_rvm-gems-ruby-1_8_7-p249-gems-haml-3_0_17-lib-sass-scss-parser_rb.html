333       # This is a nasty hack, and the only place in the parser
 
334       # that requires backtracking.
 
335       # The reason is that we can't figure out if certain strings
 
336       # are declarations or rulesets with fixed finite lookahead.
 
337       # For example, "foo:bar baz baz baz..." could be either a property
 
338       # or a selector.
 
339       #
 
340       # To handle this, we simply check if it works as a property

344       # We could eke some more efficiency out of this
 
345       # by handling some easy cases (first token isn't an identifier,
